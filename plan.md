# Project Plan: Deep Search Engine Refactoring

## 🎯 Goal (核心目标)
将当前项目重构为一个结构清晰、依赖最小化的独立“深度搜索”应用。该应用以一个外部知识库为基础，输入一个主题，输出多种格式的分析报告。

## 📍 Current Phase (当前阶段)
- [x] **Phase 0: 方案确认**
    - 重点：明确重构目标、范围和验收标准。
- [ ] **Phase 1: 结构重构与代码迁移 (MVP)**
    - 重点：建立新的目录结构，迁移并重构核心代码，替换知识库。
    - 严禁：添加新功能。
- [ ] **Phase 2: 测试与验证**
    - 重点：编写并执行测试脚本，确保核心功能（输入主题 -> 生成三种文档）通过。
- [ ] **Phase 3: 清理与交付**
    - 重点：删除所有无关文件，完善 `.gitignore` 和 `requirements.txt`。

## 🚫 Out of Scope (本阶段不做的事)
- 增加任何新的UI界面（Gradio之外）。
- 优化AI的Prompt或文档生成器的模板。
- 增加除 `.md`, `.html`, `.epub` 之外的输出格式。
